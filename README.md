# Energy System

Программный комплекс для расчёта режимов работы и энергетической эффективности оборудования ТЭС.

Программа позволяет:

- рассчитывать КПД энергоблока и ТЭС;
- учитывать температуру, влажность, скорость и направление ветра;
- выбирать режим работы: автоматический, зимний или летний;
- подбирать оптимальное разрежение в конденсаторе;
- смотреть графики и пояснение расчёта каждой точки графика;
- сохранять историю расчётов;
- экспортировать результаты в CSV и PPTX;
- вести журнал аудита действий пользователей и администраторов.

---

## 1. Требования

Рекомендуется использовать:

- Python 3.11 или новее;
- Windows 10/11, Linux или macOS;
- установленный `pip`;
- Visual Studio Code или другую IDE.

---

## 2. Установка

Откройте терминал в папке проекта.

### Windows PowerShell

```powershell
py -m venv .venv
.\.venv\Scripts\activate
pip install -r requirements.txt
```

### Linux / macOS

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Проверить установку PyQt6 можно командой:

```bash
python -c "from PyQt6.QtWidgets import QApplication; print('PyQt6 OK')"
```

---

## 3. Запуск приложения

Приложение состоит из двух частей:

1. сервер API;
2. графический интерфейс.

Их нужно запускать в двух отдельных терминалах.

### Терминал 1 — запуск API

```bash
python main.py api
```

По умолчанию API запускается по адресу:

```text
http://127.0.0.1:8000
```

Проверка API:

```text
http://127.0.0.1:8000/health
```

### Терминал 2 — запуск GUI

Во втором терминале активируйте виртуальное окружение и запустите:

```bash
python main.py gui
```

---

## 4. Первый вход

При первом запуске создаётся администратор:

```text
login: admin
```

Первичный пароль задаётся одним из двух способов.

### Вариант 1 — задать пароль заранее

Перед первым запуском API можно задать переменную окружения.

Windows PowerShell:

```powershell
$env:ENERGY_DEFAULT_ADMIN_PASSWORD="RootAdmin1!"
python main.py api
```

Linux / macOS:

```bash
export ENERGY_DEFAULT_ADMIN_PASSWORD="RootAdmin1!"
python main.py api
```

### Вариант 2 — использовать сгенерированный пароль

Если пароль заранее не задан, система сама создаст пароль и сохранит его в файл:

```text
data/initial_admin_credentials.txt
```

После первого входа программа попросит сменить первичный пароль.

---

## 5. Главное окно

В главном окне задаются параметры расчёта.

### Нагрузка и оборудование

- общая нагрузка ТЭС, МВт;
- количество работающих блоков;
- номинальная мощность блока, МВт;
- номинальный КПД блока.

### Внешние условия

- температура наружного воздуха;
- влажность;
- скорость ветра;
- направление ветра;
- режим работы:
  - `Авто`;
  - `Зимний режим`;
  - `Летний режим`.

### Дополнительные параметры

- коэффициент собственных нужд;
- коэффициент beta;
- разрежение в конденсаторе, кПа.

---

## 6. Расчёт

Нажмите кнопку:

```text
Рассчитать
```

После расчёта откроется окно результатов.

В результатах отображаются:

- применённый режим работы;
- нагрузка на энергоблок;
- КПД блока;
- КПД ТЭС брутто;
- собственные нужды;
- КПД ТЭС нетто;
- удельный расход топлива;
- мощность брутто;
- мощность нетто;
- поправочные коэффициенты;
- оптимальное количество блоков;
- оптимальное разрежение в конденсаторе.

В главном окне также появляется краткая строка последнего расчёта.

---

## 7. Графики и аудит расчёта точек

После расчёта нажмите:

```text
Графики + аудит расчета
```

Откроется окно с вкладками:

- `Аудит графиков`;
- `Температура`;
- `Блоки`;
- `Конденсатор`.

### Вкладка «Аудит графиков»

На этой вкладке видно:

- сводный график расчётных точек;
- таблицу точек графика;
- колонку `Есть trace`;
- подробное объяснение выбранной точки.

Для выбранной точки показывается:

- исходные данные;
- применённая формула;
- подстановка чисел;
- поправочные коэффициенты;
- итоговое значение расчётного ядра.

Это нужно, чтобы было понятно, что график построен не из случайных чисел, а из реального расчёта.

---

## 8. История расчётов

Нажмите кнопку:

```text
История
```

В окне истории можно посмотреть ранее выполненные расчёты.

Для каждой записи отображаются:

- дата и время;
- режим работы;
- нагрузка ТЭС;
- количество блоков;
- КПД нетто;
- удельный расход топлива.

При выборе строки ниже показываются подробные входные параметры и результат расчёта.

---

## 9. Экспорт результатов

Доступны две кнопки:

```text
Экспорт CSV
Экспорт PPTX
```

Файлы сохраняются на стороне сервера в папке:

```text
exports/
```

CSV удобно использовать для таблиц.

PPTX удобно использовать для отчёта или демонстрации результатов.

## 10. Роли пользователей

В системе есть две роли:

```text
user
admin
```

### Пользователь может:

- входить в систему;
- менять свой пароль;
- выполнять расчёты;
- смотреть результаты;
- смотреть графики;
- смотреть историю своих расчётов;
- экспортировать результаты.

### Администратор может дополнительно:

- создавать пользователей;
- назначать роли;
- смотреть журнал аудита;
- очищать старые события аудита;
- смотреть параметры политики безопасности.

Административные функции находятся в меню:

```text
Администрирование
```

---

## 11. Журнал аудита

Журнал аудита доступен администратору.

В журнал записываются:

- входы и выходы пользователей;
- неудачные попытки входа;
- блокировки учётных записей;
- расчёты;
- экспорт файлов;
- действия администратора;
- API-запросы.

Журнал не должен содержать пароли.

Окно журнала аудита позволяет:

- смотреть список событий;
- выбирать событие;
- смотреть детали события;
- очищать старые события.

---

## 12. Смена пароля

Пользователь может сменить пароль через меню:

```text
Аккаунт → Сменить пароль
```

Пароль должен соответствовать требованиям:

- содержать буквы верхнего регистра;
- содержать буквы нижнего регистра;
- содержать цифру;
- содержать разрешённый спецсимвол;
- не содержать пробелы;
- не совпадать с логином.

## 15. Частые проблемы

### PyQt6 не найден в VS Code

Если VS Code показывает ошибку:

```text
Не удается разрешить импорт "PyQt6.QtWidgets"
```

значит выбран не тот Python-интерпретатор.

Нужно выбрать интерпретатор виртуального окружения:

```text
.venv/Scripts/python.exe
```

в Windows или:

```text
.venv/bin/python
```

в Linux / macOS.

### GUI не подключается к API

Проверьте, что API запущен:

```bash
python main.py api
```

И что в браузере открывается:

```text
http://127.0.0.1:8000/health
```

### Не найден первичный пароль администратора

Проверьте файл:

```text
data/initial_admin_credentials.txt
```

Или задайте пароль через:

```text
ENERGY_DEFAULT_ADMIN_PASSWORD
```

### Экспорт не появился рядом с GUI

Экспорт сохраняется в папке сервера:

```text
exports/
```

## 16. Описание результата

После выполнения расчёта система формирует набор показателей:

| Показатель | Описание |
|---|---|
| Нагрузка на энергоблок | Распределение общей нагрузки между работающими блоками |
| КПД блока | Эффективность одного энергоблока |
| КПД ТЭС брутто | Эффективность без вычета собственных нужд |
| Собственные нужды | Внутреннее потребление мощности |
| КПД ТЭС нетто | Эффективность с учётом собственных нужд |
| Удельный расход топлива | Расход топлива на единицу выработки |
| Оптимальное разрежение | Разрежение в конденсаторе, при котором достигается лучший КПД |
| Оптимальное число блоков | Количество блоков с лучшим режимом работы |
