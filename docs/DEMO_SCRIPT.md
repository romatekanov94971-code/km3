# Сценарий демонстрации работы приложения

## 1. Установка

```bash
python -m venv .venv
source .venv/bin/activate      # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## 2. Проверка тестов

```bash
PYTHONPATH=. pytest -q
```

Ожидаемый результат:

```text
41 passed
```

## 3. Запуск API

```bash
python main.py api
```

Swagger:

```text
http://127.0.0.1:8000/docs
```

## 4. Запуск GUI

Во втором терминале:

```bash
python main.py gui
```

Первичный вход:

```text
login: admin
password: см. data/initial_admin_credentials.txt
```

Можно заранее задать пароль:

```bash
export ENERGY_DEFAULT_ADMIN_PASSWORD='RootSecure1!'
```

При первом входе система потребует сменить пароль.

## 5. Демонстрация пользовательского сценария

1. Войти в систему.
2. Ввести нагрузку ТЭС, число блоков, температуру, влажность, скорость и направление ветра.
3. Нажать «Рассчитать».
4. Показать результат:
   - нагрузка на энергоблок;
   - КПД блока;
   - КПД ТЭС брутто;
   - собственные нужды;
   - КПД ТЭС нетто;
   - удельный расход топлива.
5. Открыть график.
6. Экспортировать CSV/PPTX.

## 6. Демонстрация административного сценария

1. Открыть меню «Администрирование».
2. Создать пользователя.
3. Изменить роль пользователя.
4. Открыть «Политика безопасности».
5. Открыть «Журнал аудита».
6. Показать события входа, расчетов, создания пользователя, изменения роли, API-запросов.
7. Нажать «Очистить старые» для проверки настройки глубины хранения журналов.

## 7. Демонстрация защиты

1. Три раза ввести неверный пароль.
2. Показать блокировку учетной записи.
3. Показать, что обычный пользователь не видит меню администратора.
4. Показать, что сервер возвращает 403 при доступе пользователя к административным API.

## 8. HTTPS-режим для сетевого развертывания

```bash
export ENERGY_API_SSL_CERTFILE=certs/server.crt
export ENERGY_API_SSL_KEYFILE=certs/server.key
export ENERGY_API_BASE_URL=https://127.0.0.1:8000
python main.py api
```
