# Описание архитектуры программного комплекса

## Назначение

Система предназначена для оперативного расчета режимов работы энергообъекта, оценки эффективности энергетического оборудования ТЭС и подбора технических решений для повышения эффективности.

## Основные сценарии

1. Пользователь входит в систему.
2. При первом входе меняет первичный пароль.
3. Вводит нагрузку ТЭС, количество блоков, температуру, влажность, скорость и направление ветра.
4. Получает расчетные показатели.
5. Просматривает графики зависимости теплоотводящей способности от температуры.
6. Экспортирует результаты в CSV/PPTX.
7. Администратор создает пользователей, меняет роли, смотрит аудит и политику безопасности.

## Среда функционирования

- Язык: Python 3.
- Клиент: PyQt6.
- Сервер: FastAPI + Uvicorn.
- Хранилище: SQLite.
- Протокол взаимодействия клиента и сервера: HTTP/JSON, при сетевом развертывании HTTPS.
- Порт по умолчанию: 8000.
- Файловые каталоги: `data/`, `logs/`, `exports/`.

## Подсистемы

| Подсистема | Назначение | Основные файлы |
|---|---|---|
| Клиентская часть | GUI, ввод/вывод, графики, администрирование | `app/client/*` |
| Серверная часть | API, маршрутизация, rate limit | `app/server/*` |
| Аутентификация | Пользователи, роли, сессии, политика паролей | `app/auth/*` |
| Расчетное ядро | Формулы, расчет эффективности, оптимизация | `app/calculation/*` |
| Хранение | Пользователи, история расчетов, аудит | `app/storage/*` |
| Аудит | Регистрация событий, ротация, удаленная отправка | `app/audit/*` |
| Отчетность | CSV/PPTX, графики | `app/reporting/*` |

## Интерфейсы

| Интерфейс | Назначение |
|---|---|
| `POST /auth/login` | Вход |
| `POST /auth/logout` | Выход |
| `POST /auth/change-password` | Смена пароля |
| `GET /auth/policy` | Просмотр политики безопасности |
| `POST /auth/users` | Создание пользователя |
| `PATCH /auth/users/{username}/role` | Изменение роли |
| `POST /calc/run` | Полный расчет |
| `GET /calc/history` | История расчетов |
| `GET /calc/audit` | Журнал аудита |
| `POST /calc/audit/cleanup` | Очистка старых событий |
| `POST /calc/export/csv` | Экспорт CSV |
| `POST /calc/export/pptx` | Экспорт PPTX |

## Ограничения и указания по применению

- В локальном учебном режиме API запускается на `127.0.0.1`.
- Для сетевой эксплуатации необходимо использовать HTTPS: `ENERGY_API_SSL_CERTFILE`, `ENERGY_API_SSL_KEYFILE`.
- Модель оптимизации разрежения в конденсаторе является учебной и должна быть уточнена при наличии паспортных характеристик конкретного оборудования.
- Пароли и тела запросов с паролями не пишутся в аудит.
- Для удаленного сбора аудита задается `ENERGY_AUDIT_REMOTE_URL`.
